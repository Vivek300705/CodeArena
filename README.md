### 1пёҸвғЈ Clone the Repository

```bash
git clone https://github.com/your-username/codearena.git
cd codearena
```

### 2пёҸвғЈ Configure Environment Variables

**Backend** вҖ” create `server/.env`:

```env
# в”Җв”Җ Required в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
PORT=8000
MONGODB_URI=mongodb://localhost:27017/codearena
REDIS_URL=redis://localhost:6379
JWT_SECRET=your_super_secret_key
NODE_ENV=development

# в”Җв”Җ Optional: AI Problem Generator в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
GEMINI_API_KEY=your_gemini_api_key

# в”Җв”Җ Optional: Email (Forgot Password) в”Җв”Җв”Җв”Җв”Җ
SMTP_HOST=smtp.mailtrap.io
SMTP_PORT=2525
SMTP_EMAIL=your_email
SMTP_PASSWORD=your_password
```

**Frontend** вҖ” create `client/.env`:

```env
VITE_API_URL=http://localhost:8000
```

### 3пёҸвғЈ Build the Docker Runner Image

```bash
cd server
docker build -f Dockerfile.runner -t codearena-runner .
```

### 4пёҸвғЈ Start the Backend

```bash
cd server
npm install
npm run dev        # Development (nodemon)
# OR
npm start          # Production
```

### 5пёҸвғЈ Start the Frontend

```bash
cd client
npm install
npm run dev        # вҶ’ http://localhost:5173
```

### 6пёҸвғЈ Start the Submission Worker

```bash
# Open a separate terminal
cd server
node src/workers/submission.worker.js
```

## рҹҗі Docker Compose (Full Stack)

Run the **entire platform** вҖ” API, worker, MongoDB, Redis, and Nginx вҖ” with one command:

```bash
# Build and start all services
docker compose up --build

# Scale API horizontally (3 replicas)
docker compose up --build --scale api=3

# Run in background
docker compose up -d
```

| вҡҷпёҸ API | `8000` (internal) |
| рҹҚғ MongoDB | `27017` (internal) |

### рҹҺҜ Scoring

| Event | Points |
|---|---|
| вң… Easy problem solved | `+100 pts` |

| вҸұпёҸ Time penalty | `вҲ’2 pts/min` |

## рҹ”’ Security

| Layer | Implementation |
|---|---|
| рҹҗі **Code Sandbox** | `--cap-drop ALL`, `--read-only`, `--network none`, `--cpus=0.5`, memory-limited |
| рҹҡҰ **Rate Limiting** | Global 200 req/15min (Redis + Nginx), submissions 5/60s sliding window |
| рҹ§№ **Code Sanitization** | Blocks `$()`, backticks, null bytes вҖ” 64KB size limit |
| рҹ”‘ **Auth** | JWT (15min access) + refresh tokens in DB |
| рҹӣЎпёҸ **Headers** | Helmet вҖ” CSP, HSTS, X-Frame-Options, etc. |
| вң… **Input Validation** | Zod schemas on all endpoints |

## вҳҒпёҸ Deployment (Render)

The `render.yaml` defines two services:

| Service | Type | Description |
|---|---|---|
| рҹҢҗ `codearena-api` | Web Service | Node.js REST API |
| вҡҷпёҸ `codearena-worker` | Background Worker | Docker submission judge |

**Set these in the Render dashboard:**

```
MONGODB_URI   вҶ’  your Atlas connection string
REDIS_URL     вҶ’  your managed Redis URL
JWT_SECRET    вҶ’  auto-generated secret
```

> вҡ пёҸ **Important:** Set `USE_DOCKER=false` on Render вҖ” Docker-in-Docker is not supported. The worker runs in sandbox mode.
